# Qwen3-8B Full SFT 混合通用数据实验设计

## 背景

前一轮结果已经说明：`full_8b_ckpt350` 虽然在 DB eval 上有一定收益，但通用能力明显退化。它在 CMMLU full 从 base 的 `79.00%` 降到 `68.58%`，Multi-IF Chinese turn1/2/3 prompt strict 从 base 的 `79.30% / 65.64% / 51.10%` 降到 `56.17% / 32.82% / 14.32%`，BFCL-v3 core 补跑 `TOP_SCORE=21.28%`，其中 `AST_NON_LIVE=8.13%`、`MULTI_TURN=0.00%`。

因此下一个实验不是继续证明 full SFT 差，也不是先做 RL、拒绝式负样本或 protocol repair，而是专门验证一个更基础的问题：full-parameter SFT 是否因为训练数据过窄而发生灾难性遗忘。

## 实验目的

验证 full SFT 效果差的主因是否是窄域 DB agent 轨迹导致的通用能力遗忘，并测试混入通用中文指令/对话数据后，能否恢复 CMMLU、Multi-IF、BFCL 等通用能力，同时尽量保住 DB 优化收益。

这一轮只回答三个问题：

| 问题 | 判断依据 |
|------|----------|
| 混入通用数据能不能修复 full SFT 的通用能力退化？ | CMMLU / Multi-IF / BFCL 相比 `full_8b_ckpt350` 明显回升 |
| DB agent 数据比例应该控制在多少？ | DB eval 不明显掉，同时通用 benchmark 不再崩 |
| 如果通用能力恢复后 DB agent 轨迹仍异常，是否需要下一轮 protocol repair？ | repetition、bad tool、non-schema、finish rate、BFCL AST 是否仍异常 |

## 核心假设

| 假设 | 预期现象 | 如何验证 |
|------|----------|----------|
| `H1`：full SFT 主要问题是通用能力遗忘 | 混入高比例通用数据后，CMMLU / Multi-IF / BFCL 明显回升 | 对比 `full_8b_ckpt350` 与 mixed full SFT 的三项 benchmark |
| `H2`：DB agent 占比存在上限 | DB 占比升高时 DB eval 改善，但通用 benchmark 重新下降 | 比较 `20/80`、`25/75`、`30/70` 三个 sample ratio，并记录对应 assistant-token ratio |
| `H3`：第一轮不需要 protocol repair | 通用能力恢复后，DB rollout 的 repetition / bad tool 同步下降或保持可控 | 若仍异常，再进入第二轮 protocol repair |

## 数据设计

| 数据桶 | 来源 | 类型 | 第一轮用途 |
|--------|------|------|------------|
| `db_agent` | 本项目已有 DB-Opt SFT 轨迹 | DB agent 多轮轨迹 | 保留任务能力和 TPS 收益 |
| `general_chat` | `shareAI/ShareGPT-Chinese-English-90k`；备选 `FreedomIntelligence/sharegpt-chinese` | 通用中英/中文多轮对话 | 防 full SFT 遗忘的主数据 |
| `general_inst` | `m-a-p/COIG-CQIA`；备选 `BelleGroup/train_3.5M_CN` | 中文通用指令 | 补中文问答、解释、写作、代码、知识类指令 |

数据选择注意：

| 数据集 | 当前判断 | 使用策略 |
|--------|----------|----------|
| `shareAI/ShareGPT-Chinese-English-90k` | Apache-2.0，真实 ShareGPT 风格中英对话 | 第一优先级 |
| `FreedomIntelligence/sharegpt-chinese` | Apache-2.0，中文 ShareGPT 翻译数据 | 作为 `general_chat` 备选或补充 |
| `COIG-CQIA` | 高质量中文指令，但 HF 页面 license 信息不完整 | license 确认可用后再进入主训练 |
| `BELLE train_3.5M_CN` | GPL-3.0 且有研究/非商业限制 | 只作为研究备选，不作为默认主数据 |

## 数据预处理

| 数据桶 | 处理 |
|--------|------|
| `db_agent` | 使用现有 assistant-only loss 口径；过滤明显坏 schema、空输出、严重截断、明显重复 knob 的异常样本；不引入失败 rollout 纠偏样本 |
| `general_chat` | 转成统一 `messages` 格式；保留多轮；只对 assistant 计算 loss；过滤过长、空回复、明显广告/脏文本 |
| `general_inst` | 转成单轮或多轮 `messages` 格式；只对 assistant 计算 loss；过滤与 benchmark 测试集高度相似的模板化选择题 |
| benchmark 数据 | 不得把 CMMLU、Multi-IF、BFCL、DB eval 测试场景加入训练 |

通用数据先构建一个可复用 pool，不代表第一轮训练会全部吃进去。第一轮实验按 sample ratio 设定，更容易对齐 AgentTuning / Agent-FLAN 等 agent 训练论文的表述；同时必须记录实际 assistant-token ratio，避免 DB agent 轨迹更长导致监督权重被误读。

## Sample 与 Token 比例映射

当前候选池统计口径：使用 Qwen3-8B tokenizer，统计 assistant-only SFT 中实际产生监督的 assistant tokens。

| 数据池 | rows | assistant tokens mean | p50 | p90 | p95 |
|--------|-----:|----------------------:|----:|----:|----:|
| `db_agent` | 6,645 | 799.8 | 783 | 985 | 1,070 |
| `general_pool` | 48,650 | 386.4 | 278 | 838 | 995 |

在上述候选池下，如果使用全量 `db_agent`，目标 DB assistant-token 占比与所需 general 条数的对应关系如下：

| 目标 DB token 占比 | 需要 general 条数 | 等价 DB sample 占比 | 是否可用全量 DB |
|-------------------:|------------------:|--------------------:|----------------|
| 10% | 123,780 | 5.09% | 不可，general 不够 |
| 20% | 55,013 | 10.78% | 不可，general 不够 |
| 30% | 32,091 | 17.15% | 可以 |
| 40% | 20,630 | 24.36% | 可以 |
| 50% | 13,753 | 32.58% | 可以 |

反过来，如果按 sample ratio 设实验，并使用全量 `db_agent`，大致映射如下：

| DB sample 占比 | general 条数 | 对应 DB assistant-token 占比 |
|---------------:|-------------:|-----------------------------:|
| 10% | 59,805 | 25.67% |
| 15% | 37,655 | 28.89% |
| 20% | 26,580 | 34.09% |
| 25% | 19,935 | 40.78% |
| 30% | 15,505 | 47.03% |
| 40% | 9,968 | 58.00% |
| 50% | 6,645 | 67.43% |

结论：sample ratio 和 token ratio 在语义上可以互相换算。第一轮不再纠结“必须按 token 还是 sample”，而是按 sample ratio 设计实验，并在实验记录里同步报告实际 assistant-token ratio。

## 第一轮实验矩阵

| 实验名 | sample ratio | 约等价 DB assistant-token ratio | general 条数 | general 内部组成 | 目的 |
|--------|-------------:|-------------------------------:|-------------:|------------------|------|
| `full_mix_s20_db20_gen80` | 20% `db_agent` + 80% `general` | 34.09% | 26,580 | `70% general_chat + 30% general_inst` | 主实验；对齐 AgentTuning 高通用比例，同时 DB token 权重仍不低 |
| `full_mix_s25_db25_gen75` | 25% `db_agent` + 75% `general` | 40.78% | 19,935 | `70% general_chat + 30% general_inst` | 中间档；观察 DB 收益和遗忘的折中 |
| `full_mix_s30_db30_gen70` | 30% `db_agent` + 70% `general` | 47.03% | 15,505 | `70% general_chat + 30% general_inst` | 探索 DB agent 占比上限 |

执行顺序：

| 顺序 | 实验 | 是否必须 | 继续条件 |
|------|------|----------|----------|
| 1 | `full_mix_s20_db20_gen80` | 必跑 | 如果 CMMLU 回到 `77%+` 且 DB avg 不低于 `4%`，它就是主候选 |
| 2 | `full_mix_s25_db25_gen75` | 必跑 | 如果 `s20` DB 收益不足，检查提高 DB 占比是否仍不遗忘 |
| 3 | `full_mix_s30_db30_gen70` | 条件跑 | 只有当 `s20/s25` 通用能力恢复但 DB 收益不足时再跑 |

### 1. 构建混合训练集

该命令会从全量 `db_agent` 加上已下载的 `general_pool` 候选数据中，按上表 sample ratio 生成三个训练集：

| 输出目录 | DB rows | general rows |
|----------|--------:|-------------:|

```bash
cd /root/workspace/DB-Opt-R1

PROJECT_ROOT=/root/workspace/DB-Opt-R1 \
TOKENIZER_PATH=/root/workspace/models/Qwen3-8B \
DATA_ROOT=/root/workspace/datasets/agent_length_refs \
python - <<'PY'
import os, json, random
from pathlib import Path
from statistics import mean
from transformers import AutoTokenizer

ROOT = Path(os.environ["PROJECT_ROOT"])
DATA_ROOT = Path(os.environ["DATA_ROOT"])
TOKENIZER_PATH = os.environ["TOKENIZER_PATH"]
OUT_ROOT = ROOT / "data_pipeline/data/train/v2/full_sft_general_mix"
DB_PATH = ROOT / "data_pipeline/data/train/v2/sft_trajectories_v10_train.jsonl"

SEED = 42
TARGETS = {
    "full_mix_s20_db20_gen80": 26580,
    "full_mix_s25_db25_gen75": 19935,
    "full_mix_s30_db30_gen70": 15505,
}
GENERAL_ROOTS = [
    DATA_ROOT / "AgentInstruct/data",
    DATA_ROOT / "Agent-FLAN/data",
    DATA_ROOT / "hermes-function-calling-v1",
]

tok = AutoTokenizer.from_pretrained(TOKENIZER_PATH, trust_remote_code=True, local_files_only=True)
rng = random.Random(SEED)

ROLE_MAP = {
    "human": "user",
    "user": "user",
    "prompter": "user",
    "gpt": "assistant",
    "assistant": "assistant",
    "bot": "assistant",
    "chatgpt": "assistant",
    "system": "system",
    "function": "tool",
    "tool": "tool",
    "observation": "tool",
}

def clean(x):
    if x is None:
        return ""
    if isinstance(x, (dict, list)):
        x = json.dumps(x, ensure_ascii=False)
    return str(x).replace("\r\n", "\n").replace("\r", "\n").strip()

def as_list(x):
    if x is None:
        return None
    if isinstance(x, list):
        return x
    if isinstance(x, tuple):
        return list(x)
    if hasattr(x, "tolist"):
        return x.tolist()
    return None

def role(x):
    if x is None:
        return ""
    return ROLE_MAP.get(str(x).strip().lower(), str(x).strip().lower())

def add(msgs, r, content, loss=None):
    r = role(r)
    content = clean(content)
    if r not in {"system", "user", "assistant"} or not content:
        return
    msg = {"role": r, "content": content}
    if loss is not None:
        msg["loss"] = bool(loss)
    msgs.append(msg)

def normalize_record(rec, source_dataset):
    msgs = []
    messages = rec.get("messages")
    if as_list(messages) is not None:
        for m in as_list(messages):
            if isinstance(m, dict):
                add(msgs, m.get("role") or m.get("from"), m.get("content") or m.get("value") or m.get("text"), m.get("loss"))
    else:
        conv = rec.get("conversations")
        if conv is None:
            conv = rec.get("conversation")
        conv_list = as_list(conv)
        if conv_list is not None:
            for m in conv_list:
                if isinstance(m, dict):
                    add(msgs, m.get("role") or m.get("from"), m.get("content") or m.get("value") or m.get("text"), m.get("loss"))
        elif isinstance(conv, dict):
            roles = as_list(conv.get("from") or conv.get("role"))
            vals = as_list(conv.get("value") or conv.get("content") or conv.get("text"))
            losses = as_list(conv.get("loss"))
            if roles is not None and vals is not None:
                for i, (r, v) in enumerate(zip(roles, vals)):
                    loss = losses[i] if losses is not None and i < len(losses) else None
                    add(msgs, r, v, loss)
        else:
            prompt = rec.get("instruction") or rec.get("prompt") or rec.get("question")
            output = rec.get("output") or rec.get("answer") or rec.get("response") or rec.get("answers")
            if prompt and output:
                add(msgs, "user", prompt)
                add(msgs, "assistant", output, True)

    while msgs and msgs[0]["role"] == "assistant":
        msgs.pop(0)
    while msgs and msgs[-1]["role"] != "assistant":
        msgs.pop()
    if not msgs or not any(m["role"] == "user" for m in msgs) or not any(m["role"] == "assistant" for m in msgs):
        return None

    return {
        "messages": msgs,
        "data_source": "general",
        "source_dataset": source_dataset,
        "enable_thinking": False,
    }

def iter_jsonl(path):
    with path.open("r", encoding="utf-8", errors="replace") as f:
        for line in f:
            if not line.strip():
                continue
            try:
                yield json.loads(line)
            except Exception:
                continue

def iter_json(path):
    try:
        data = json.loads(path.read_text(encoding="utf-8", errors="replace"))
    except Exception:
        return
    if isinstance(data, list):
        yield from data
    elif isinstance(data, dict):
        for k in ("data", "train", "records"):
            if isinstance(data.get(k), list):
                yield from data[k]
                return
        yield data

def iter_parquet(path):
    import pandas as pd
    for item in pd.read_parquet(path).to_dict(orient="records"):
        yield item

def iter_records(root):
    root = Path(root)
    files = [root] if root.is_file() else sorted(
        p for p in root.rglob("*")
        if p.suffix in {".jsonl", ".json", ".parquet"} and ".hfd" not in p.parts
    )
    for p in files:
        if p.suffix == ".jsonl":
            yield from iter_jsonl(p)
        elif p.suffix == ".json":
            yield from iter_json(p)
        elif p.suffix == ".parquet":
            yield from iter_parquet(p)

def assistant_tokens(row):
    text = "\n".join(
        m.get("content", "")
        for m in row.get("messages", [])
        if m.get("role") == "assistant" and m.get("loss", True) is not False
    )
    return len(tok.encode(text, add_special_tokens=False)) if text else 0

def load_jsonl(path):
    with path.open("r", encoding="utf-8") as f:
        return [json.loads(line) for line in f if line.strip()]

db_rows = load_jsonl(DB_PATH)
for row in db_rows:
    row.setdefault("data_source", "db_agent")

general_rows = []
seen = set()
for root in GENERAL_ROOTS:
    if not root.exists():
        continue
    for rec in iter_records(root):
        row = normalize_record(rec, source_dataset=root.parent.name if root.name == "data" else root.name)
        if row is None:
            continue
        key = json.dumps(row["messages"], ensure_ascii=False, sort_keys=True)
        if key in seen:
            continue
        seen.add(key)
        general_rows.append(row)

if len(general_rows) < max(TARGETS.values()):
    raise RuntimeError(f"general rows 不足: {len(general_rows)} < {max(TARGETS.values())}")

rng.shuffle(general_rows)

for exp, n_general in TARGETS.items():
    out_dir = OUT_ROOT / exp
    out_dir.mkdir(parents=True, exist_ok=True)
    selected_general = general_rows[:n_general]
    rows = list(db_rows) + list(selected_general)
    rng.shuffle(rows)

    train_path = out_dir / "train.jsonl"
    stats_path = out_dir / "stats.json"
    with train_path.open("w", encoding="utf-8") as f:
        for row in rows:
            f.write(json.dumps(row, ensure_ascii=False) + "\n")

    db_tokens = [assistant_tokens(row) for row in db_rows]
    gen_tokens = [assistant_tokens(row) for row in selected_general]
    stats = {
        "experiment": exp,
        "seed": SEED,
        "train_file": str(train_path),
        "db_rows": len(db_rows),
        "general_rows": len(selected_general),
        "total_rows": len(rows),
        "db_assistant_tokens_mean": mean(db_tokens),
        "general_assistant_tokens_mean": mean(gen_tokens),
        "db_assistant_tokens_sum": sum(db_tokens),
        "general_assistant_tokens_sum": sum(gen_tokens),
        "db_assistant_token_ratio": sum(db_tokens) / (sum(db_tokens) + sum(gen_tokens)),
    }
    stats_path.write_text(json.dumps(stats, ensure_ascii=False, indent=2) + "\n", encoding="utf-8")
    print(json.dumps(stats, ensure_ascii=False, indent=2))
PY
```

## 训练设置

| 项 | 设置 |
|----|------|
| 训练方式 | full SFT，`trl SFTTrainer` 全参数微调 |
| loss mask | assistant-only，与前面实验保持一致 |
| learning rate | 第一轮使用 `1e-6`，优先防遗忘；若 DB 学习不足再考虑 `3e-6` |
| epoch | 先跑 `1 epoch` |
| checkpoint | 保存 `1/3 epoch`、`2/3 epoch`、`1 epoch` |
| 混合方式 | 按 sample ratio 采样，同时记录实际 assistant-token ratio；当前 `20%/25%/30%` DB sample 约对应 `34%/41%/47%` DB assistant-token |
| 最大长度 | 与现有 SFT 训练口径保持一致；超长 general 样本截断或过滤，不影响 DB 轨迹格式 |

## 评测口径

| 评测 | 指标 | 目标 |
|------|------|------|
| DB eval | 三 baseline 报表 | avg vs 场景原始配置至少 `4%+`，理想接近或超过 `full_8b_ckpt350` 的 `5.45%` |
| 轨迹稳定性 | repetition、trunc/protocol、bad_tool_rows、non_schema_rows、predict_rate、finish_rate | 明显优于当前 full SFT 崩坏 checkpoint，至少不能劣于 `full_d2_3k_lr1e5_ep3_ckpt150` |
| CMMLU full | `OVERALL mean_acc` | 从 `68.58%` 回升到 `77%+` |
| Multi-IF Chinese | turn1/2/3 prompt strict | 不再出现 `56.17/32.82/14.32` 级别的腰斩 |
| BFCL-v3 core | `TOP_SCORE`、`AST_NON_LIVE`、`MULTI_TURN`、`RELEVANCE` | AST 不应停留在 `8%` 崩坏区间，multi-turn 不应为 `0%` |

## 决策规则

| 结果 | 解释 | 下一步 |
|------|------|--------|
| `s20` 通用恢复，DB 收益低 | 通用数据有效，但 DB 占比不足 | 看 `s25/s30` |
| `s20` 通用恢复且 DB avg `4-5%+` | `20/80` sample 已经是可用主线 | 围绕 `s20` 微调数据质量和 LR |
| `s25/s30` DB 更好但通用明显下降 | DB 占比上限低于对应档位 | 回到前一个稳定档 |
| `s20/s25` 都不能恢复 CMMLU | 不是单纯比例问题，可能是 LR、通用数据质量或格式处理问题 | 降 LR 到 `5e-7` 或更换通用数据 |
| 通用恢复但 BFCL/DB 协议仍崩 | 遗忘得到缓解，但工具协议能力仍有单独问题 | 第二轮再设计 `protocol_repair` 或少量 function-calling 补丁 |

## 第一轮不做

| 不做 | 原因 |
|------|------|
| 不引入 RL | 当前任务是修 full SFT 遗忘，RL 会放大奖励偏置，干扰根因判断 |
| 不大规模加入 ToolBench/xLAM/Glaive/Hermes | 它们主要是 tool/function calling 专项数据，不是通用能力保底数据 |
| 不先做拒绝式 negative samples | DB-Opt eval 基本都是 DB 调优场景，不存在开放域“拒绝用户”的主边界 |
| 不先做 protocol repair | 当前主证据是通用能力和 function calling 能力整体退化，不是单一失败轨迹幻觉 |
