# SFT 轨迹数据合成方案

## 背景

GRPO 训练前需要 SFT 冷启动，让模型学会工具调用格式和基本调优策略。本文档梳理业界 Agent 轨迹数据合成方案，并选定适合本项目的方案。

## 方案对比

### 方案 1：Teacher Distillation（教师蒸馏）

用强模型（GPT-5）在环境中执行完整 episode，收集成功轨迹。

```
GPT-5 + 模拟环境 → 生成 N 条轨迹 → 过滤 reward > 0 的 → SFT 数据
```

**轨迹格式**：
```
System: 你是 PostgreSQL 调优专家，使用以下工具优化数据库...
User: 请优化这个数据库的性能
Assistant: <think>先看硬件和当前指标</think>
<tool_call>{"name": "get_hardware_info", "arguments": {}}</tool_call>
Observation: {"cpu_count": 8, "total_memory_gb": 32, ...}
Assistant: <think>32G 内存，shared_buffers 可以调大...</think>
<tool_call>{"name": "set_knob", "arguments": {"knobs": "{\"shared_buffers\": \"8GB\"}"}}</tool_call>
Observation: {"success": ["shared_buffers"], "pending_restart": [...]}
...
```

| 优点 | 缺点 |
|------|------|
| 轨迹质量高（GPT-5 水平） | 需要调用 API，有成本 |
| 能利用我们的模拟环境 | 依赖 GPT-5 对 PG 调优的了解程度 |
| 自动化程度高 | 生成速度受 API 限制 |

**参考**：大多数 Agent SFT 论文使用此方案

---

### 方案 2：Simia-SFT（种子扩展）

手写几条种子轨迹，用 LLM 模拟环境反馈来扩展出大量新轨迹。

```
5-10 条种子轨迹 → LLM 同时扮演 Agent 和 Environment → 扩展 1000+ 条 → 规则校验
```

| 优点 | 缺点 |
|------|------|
| 不需要真实环境 | 环境反馈不真实（LLM 编造的） |
| 少量种子即可扩展 | 需要设计复杂的 prompt |
| 成本较低 | 多样性有限 |

**参考**：Simia（arXiv:2511.01824）

---

### 方案 3：ReTool Cold-start（冷启动 SFT）

让 LLM 生成带工具调用的长推理链，不真实执行工具，只学格式和推理过程。

```
Prompt: "给定这个环境，写出完整的调优推理过程（包含工具调用）"
→ LLM 生成完整 trace（含 <tool_call> 和 模拟的 observation）
→ SFT 训练
→ GRPO 进一步训练
```

| 优点 | 缺点 |
|------|------|
| 最简单，不需要环境交互 | 工具返回是 LLM 编的，不真实 |
| 适合冷启动 | 模型只学了格式，没学到真实策略 |
| 成本最低 | 需要 GRPO 来弥补质量差距 |

**参考**：ReTool（arXiv:2504.09649）

---

### 方案 4：STeP（带纠错的轨迹）

生成包含错误→反思→纠正的轨迹，让模型学会 self-correction。

```
正常轨迹：观察 → 合理设置 → 性能提升
纠错轨迹：观察 → 错误设置 → 性能下降 → <reflect>shared_buffers 设太大了</reflect> → 纠正 → 提升
```

训练时 mask 掉错误步骤，只学反思和纠正。

| 优点 | 缺点 |
|------|------|
| 模型学会自我纠错 | 构造成本高 |
| 更鲁棒 | 需要标注错误类型 |

**参考**：STeP（arXiv）

---

### 方案 5：Agent-R（MCTS 搜索）

用 Monte Carlo Tree Search 搜索最优调优路径，同时收集失败分支。

```
             root（默认配置）
            /     \
      调 shared_buffers   调 work_mem
        /    \              /    \
   restart  reload     restart  reload
     ↓        ↓          ↓        ↓
  +15% tps  +8% tps   +5% tps  +12% tps
```

选最优路径做正样本，差路径做负样本。

| 优点 | 缺点 |
|------|------|
| 能找到最优策略 | 搜索空间大，成本高 |
| 有正负对比数据 | 需要大量 cost model 调用 |

**参考**：Agent-R（arXiv）

---

## 推荐方案

结合我们的条件（有模拟环境 + Cost Model + 预算有限），推荐分两阶段：

### 阶段 1：冷启动 SFT（方案 3）

用 GPT-5 生成 200-500 条完整推理轨迹（含工具调用），不经过真实环境。目的：让模型学会：
- 工具调用格式（`<tool_call>...</tool_call>`）
- 基本调优思路（先观察 → 分析 → 设 knob → 验证）
- 推理链风格（`<think>...</think>`）

### 阶段 2：环境交互 SFT（方案 1）

用 GPT-5 在模拟环境中跑完整 episode，过滤高 reward 轨迹。目的：让模型从真实工具反馈中学到有效策略。

### 阶段 3：GRPO 训练

在模拟环境中用 GRPO 进一步优化。

```
阶段 1（冷启动 SFT）→ 阶段 2（环境 SFT）→ 阶段 3（GRPO）
       学格式              学策略              优化策略
```
