# MCTS 轨迹合成设计

## 概述

使用 Monte Carlo Tree Search 在模拟环境中搜索最优调优路径，生成高质量 SFT 轨迹数据。每个节点是一个 Agent 决策点，边是工具调用，叶节点用 Cost Model 评估性能。

## 核心思想

```
传统方法：GPT-5 贪心决策 → 一条轨迹
MCTS 方法：在每个决策点展开多个分支 → 搜索后选最优路径
```

好处：
1. **找到更优策略**：不依赖单次贪心，充分探索
2. **正负样本对比**：最优路径做正样本，差路径做负样本
3. **可控多样性**：不同搜索产生不同策略风格

## 搜索树结构

```
Root（环境初始状态）
│
├── [观察] get_hardware_info → get_db_metrics
│   │
│   ├── [行动A] set_knob(shared_buffers=4GB)
│   │   ├── restart_pg → predict  → reward=+8%   ← 中等
│   │   └── set_knob(work_mem=256MB) → predict → reward=+12%  ← 好
│   │
│   ├── [行动B] set_knob(shared_buffers=8GB)
│   │   ├── restart_pg → predict  → reward=+15%  ← 最优 ✓
│   │   └── set_knob(work_mem=64MB) → predict → reward=+6%
│   │
│   └── [行动C] set_knob(work_mem=512MB)
│       └── predict → reward=+3%                  ← 差
│
└── [观察] get_workload_info → get_recent_logs
    └── ...
```

## MCTS 四个阶段

### 1. Selection（选择）

从根节点沿 UCB1 分数最高的路径向下选择，直到找到未完全展开的节点。

```python
def select(node):
    while node.is_fully_expanded():
        node = max(node.children, key=ucb1_score)
    return node

def ucb1_score(node):
    exploit = node.total_reward / node.visit_count
    explore = C * sqrt(log(node.parent.visit_count) / node.visit_count)
    return exploit + explore
```

`C` 控制探索-利用平衡，推荐 `C = 1.414`。

### 2. Expansion（展开）

在选中节点生成新的子节点。用 LLM 生成该步骤的候选行动。

```python
def expand(node, llm):
    # 构造 prompt：当前轨迹 + "你的下一步操作是什么？"
    prompt = build_prompt(node.trajectory_so_far)

    # LLM 生成 K 个候选行动（temperature > 0）
    candidates = []
    for _ in range(K):
        action = llm.generate(prompt, temperature=0.8)
        candidates.append(action)

    # 去重，创建子节点
    for action in deduplicate(candidates):
        child = Node(action=action, parent=node)
        node.children.append(child)

    return random.choice(node.children)  # 返回一个未访问的子节点
```

### 3. Simulation（模拟）

从新节点出发，在模拟环境中执行到 episode 结束，获取 reward。

```python
def simulate(node, env, llm):
    # 在模拟环境中执行当前轨迹
    env.reset(sample_idx=node.root.sample_idx)
    replay_actions(env, node.trajectory_so_far)

    # 继续 rollout 直到结束
    for _ in range(remaining_turns):
        action = llm.generate(build_prompt(env.history), temperature=0.3)
        obs, reward, done, info = step(env, action)
        if done:
            break

    # 最终 reward = cost model 预测的性能提升
    return compute_final_reward(env)
```

### 4. Backpropagation（回传）

将 reward 沿路径回传到根节点。

```python
def backpropagate(node, reward):
    while node is not None:
        node.visit_count += 1
        node.total_reward += reward
        node = node.parent
```

## 节点定义

```python
class MCTSNode:
    def __init__(self, action=None, parent=None):
        self.action = action            # 这一步的工具调用（str）
        self.observation = None         # 工具返回（str）
        self.parent = parent
        self.children = []
        self.visit_count = 0
        self.total_reward = 0.0

    @property
    def trajectory_so_far(self):
        """从根到当前节点的完整轨迹"""
        path = []
        node = self
        while node.parent is not None:
            path.append((node.action, node.observation))
            node = node.parent
        return list(reversed(path))

    def is_fully_expanded(self):
        return len(self.children) >= MAX_CHILDREN and all(
            c.visit_count > 0 for c in self.children
        )
```

## 轨迹提取

搜索完成后，从树中提取 SFT 数据：

### Best-in-Tree（最优路径）
```python
def extract_best_trajectory(root):
    """沿平均 reward 最高的路径提取"""
    path = []
    node = root
    while node.children:
        node = max(node.children, key=lambda c: c.total_reward / max(c.visit_count, 1))
        path.append({"action": node.action, "observation": node.observation})
    return path
```

### 正负样本对比
```python
def extract_contrastive_pairs(root):
    """提取最优和最差的子路径做对比"""
    pairs = []
    for node in all_decision_points(root):
        if len(node.children) < 2:
            continue
        children_sorted = sorted(node.children,
            key=lambda c: c.total_reward / max(c.visit_count, 1), reverse=True)
        pairs.append({
            "context": node.trajectory_so_far,
            "chosen": children_sorted[0].action,     # 好的选择
            "rejected": children_sorted[-1].action,   # 差的选择
        })
    return pairs
```

## 参数设置

| 参数 | 值 | 说明 |
|------|-----|------|
| `num_simulations` | 50 | 每棵树的 MCTS 迭代次数 |
| `max_children` (K) | 3-5 | 每个节点最多展开子节点数 |
| `max_depth` | 10 | 最大轨迹深度（=max_turns） |
| `C` (UCB1) | 1.414 | 探索系数 |
| `expand_temperature` | 0.8 | 展开时 LLM 采样温度 |
| `rollout_temperature` | 0.3 | 模拟时 LLM 采样温度（更贪心） |
| `num_environments` | 100-500 | 搜索的环境数（数据集样本数） |

## 完整流程

```python
dataset = load_dataset("cost_model/data/raw/dataset.csv")
cost_model = load_model("outputs/cost_model/best.pkl")
llm = load_llm("gpt-4")  # 或本地模型

sft_data = []
contrastive_data = []

for sample_idx in range(len(dataset)):
    # 1. 建树
    env = DBToolEnv(mode="train", dataset_path=..., cost_model=cost_model)
    root = MCTSNode()
    root.sample_idx = sample_idx

    # 2. MCTS 搜索
    for _ in range(num_simulations):
        leaf = select(root)
        child = expand(leaf, llm)
        reward = simulate(child, env, llm)
        backpropagate(child, reward)

    # 3. 提取轨迹
    best_traj = extract_best_trajectory(root)
    sft_data.append(format_as_conversation(best_traj))

    pairs = extract_contrastive_pairs(root)
    contrastive_data.extend(pairs)

# 4. 保存
save_jsonl(sft_data, "data/sft_trajectories.jsonl")
save_jsonl(contrastive_data, "data/contrastive_pairs.jsonl")
```

## 成本估算

假设用 GPT-5 作为搜索用 LLM：

| 项目 | 计算 |
|------|------|
| 每次 expand | ~3-5 次 LLM 调用 |
| 每次 simulate | ~5-10 次 LLM 调用（rollout） |
| 每棵树 50 次迭代 | ~50 × 10 = 500 次 LLM 调用 |
| 100 个环境 | ~50,000 次 LLM 调用 |
| GPT-5 成本 | ~$150-300（取决于 token 长度） |

如果用本地模型（如 Qwen-72B）做搜索，成本约为 GPU 时间 ~20-50 小时。
