# 实验 01：MLP 模型规模对 Cost Model 精度的影响

## 实验目标

验证 MLP 模型容量（参数量）是否是 Cost Model 在 1K+ TPS 区间精度瓶颈。

## 实验现状

- 当前 MLP `[512,256,128,64]`（~230K 参数），1K+ 区间训练集 MAE ~28%，验证集 MAE ~52%
- LightGBM 过拟合实验（256 叶/500 轮/无正则）可以将训练集 MAE 降到 0%
- MLP 是否也能通过扩大模型规模降低误差，尚未验证

## 实验设计

### 数据集
- `all_with_active.json`（12829 条，含 IO 特征）
- 特征 122 个（含悬崖阈值特征）

### 模型规模梯度

| 编号 | 架构 | 参数量（约） |
|---|---|---|
| S1 | [64, 32] | ~10K |
| S2 | [256, 128] | ~40K |
| S3 | [512, 256, 128, 64] | ~230K |
| S4 | [1024, 512, 256, 128] | ~900K |
| S5 | [2048, 1024, 512, 256] | ~3.5M |

固定配置：`dropout=0.1, BatchNorm, ReLU, lr=1e-3, AdamW, CosineAnnealing, batch=512`

### 阶段一：记忆能力

- 全量数据训练（不拆 validation），关闭 early stopping，跑满 1000 epochs
- 衡量标准：训练 MAE < 5% 视为"已记住"

### 阶段二：泛化能力

- 80/20 split（`random_state=42`），开启 early stopping（patience=30），max 300 epochs
- 记录训练集 MAE 和验证集 MAE

## 预期结论判断

| 阶段一 | 阶段二 | 结论 |
|---|---|---|
| 小模型记不住，大模型能记住 | 大模型验证 MAE 也更好 | 模型太小，应扩容 |
| 小模型记不住，大模型能记住 | 大模型验证 MAE 没改善 | 过拟合问题，不是容量 |
| 所有模型都能记住 | 验证 MAE 都差不多 | 瓶颈不在模型 |
| 所有模型都记不住 | — | 特征表达力不足 |

## 实验结果（WAPE 指标）

> WAPE = Σ|pred - actual| / Σ actual × 100%

### 阶段一：记忆能力（全量训练 1000 epochs）

| 编号 | 架构 | 参数量 | 全局 WAPE | 1K+ WAPE | 是否记住(<5%) |
|---|---|---|---|---|---|
| S1 | [64,32] | 10K | 27.5% | 27.9% | ❌ |
| S2 | [256,128] | 65K | 18.0% | 18.2% | ❌ |
| S3 | [512,256,128,64] | 237K | 6.3% | 6.3% | ❌ |
| S4 | [1024,512,256,128] | 819K | 5.6% | 5.7% | ≈ |
| S5 | [2048,1024,512,256] | 3.0M | 3.6% | 3.6% | ✅ |
| S6 | [4096,2048,1024,512] | 11.5M | 3.5% | 3.5% | ✅ |

### 阶段二：泛化能力（80/20 split + early stopping）

| 编号 | best_ep | 验证全局 WAPE | 验证 1K+ WAPE |
|---|---|---|---|
| S1 | 295 | 47.0% | 47.4% |
| S2 | 291 | 46.0% | 46.3% |
| **S3** | **234** | **34.9%** | **35.2%** ← 最优 |
| S4 | 225 | 36.7% | 37.0% ↑ |
| S5 | 171 | 38.2% | 38.5% ↑ |
| S6 | 188 | 41.5% | 41.7% ↑ |

### 结论分析

#### 观察 1：模型有能力记住数据

S5(3M) 和 S6(11.5M) 均能将训练 WAPE 降至 3.5%，说明特征维度足以区分样本。

#### 观察 2：S3 是最优模型规模，S4+ 开始过拟合

验证 1K+ WAPE 趋势：

- S1(47.4%) → S2(46.3%) → **S3(35.2%)** → S4(37.0%) → S5(38.5%) → S6(41.7%)

**S3 是拐点**。S3 之后验证 WAPE 持续上升（35.2% → 41.7%），这是过拟合的标志——模型容量超过 237K 后，新增参数用于记忆训练噪声，验证误差反而恶化。

同时注意 best_ep 从 S1(295) 持续降至 S5(171)，说明大模型 early stopping 更早触发，也印证了大模型更容易过拟合。

#### 最终结论

**当前 baseline S3（237K 参数）已经是最优模型规模。** 增大参数量无法改善泛化，反而导致过拟合。模型容量不是瓶颈。
