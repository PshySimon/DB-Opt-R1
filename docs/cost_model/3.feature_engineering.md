# Cost Model 特征工程设计

## 一、概述

本文档定义 Cost Model 的特征工程方案：从原始采集数据到模型输入的完整预处理流程。

数据来源：`cost_model/data/raw/dataset.csv`（多机器、多负载采集合并）。

---

## 二、原始数据结构

采集 pipeline（`collector.py`）输出 CSV 包含 5 类列：

| 前缀 | 数量 | 说明 |
|------|------|------|
| `hw_*` | ~9 | 硬件特征 |
| `sw_*` | ~8 | 软件特征 |
| `knob_*` | ~363 | SHOW ALL 的所有 PG 参数 |
| `metric_*` | ~81 | pg_stat 视图运行时指标差值 |
| 其他 | 6 | timestamp, status, error, workload, tps, latency_avg, latency_p95 |

---

## 三、特征筛选

### 3.1 删除的列

| 列 | 原因 |
|----|------|
| `timestamp` | 无预测意义 |
| `sw_*`（全部 8 列） | 同一机器不变，跨机器由 hw_* 替代 |
| `metric_*`（全部 ~81 列） | 预测时拿不到（跑完 benchmark 才有） |
| `knob_*` 中不变化的（~316 列） | 所有行相同，无方差 |
| `shared_memory_size` | shared_buffers 的衍生量，相关系数 0.999 |
| `shared_memory_size_in_huge_pages` | 同上 |
| `hw_disk_read/write_bandwidth_mbps` | 未采集，全部为空 |
| `huge_pages`（如果只有一个值） | 无方差 |
| `status`, `error` | 辅助字段，不作为输入特征 |

### 3.2 保留的列

最终保留 **~45 个输入特征 + 1 个目标**：

```
47 个变化 knob - 2 个衍生量 = 45 个 knob
+ 5 个硬件数值特征 (hw_*)
+ 1 个负载类型 (workload → one-hot 4 列)
= ~54 个输入特征

目标: tps → log1p(tps)
```

---

## 四、缺失值处理

| 场景 | 缺失原因 | 处理方式 |
|------|---------|---------|
| 成功行 knob_* | 不会缺失（SHOW ALL 全量） | — |
| 失败行 knob_* | `_build_fail_row` 仅记录设置的 knob | 用 `knob_space.yaml` 中的 **default** 值填充 |
| `hw_available_memory_gb` | 每轮略有波动 | 正常，保留 |
| `hw_disk_*_bandwidth` | 未用 fio 采集 | 删除 |
| `latency_p95` | pgbench 默认不输出 | 可为 None，不影响 TPS 预测 |

---

## 五、特征变换

### 5.1 内存型 Knob → 统一为 MB + log2

PG 内存参数带单位（`"128MB"`, `"2GB"`），跨越 MB~GB 量级。

```python
def parse_memory_to_mb(val: str) -> float:
    """'128MB' → 128.0, '2GB' → 2048.0, '8kB' → 0.0078"""
    # 解析数值 + 单位，统一转换为 MB
```

变换：`x → log2(max(x_mb, 1))`

适用：shared_buffers, work_mem, effective_cache_size, maintenance_work_mem, wal_buffers, temp_buffers, max_wal_size, min_wal_size

> 例：shared_buffers 范围 32MB~8GB → log2 后变为 5~13，分布更均匀。

### 5.2 时间型 Knob → 统一为 ms + log10

PG 时间参数带单位（`"200ms"`, `"5min"`）。

```python
def parse_time_to_ms(val: str) -> float:
    """'200ms' → 200.0, '5min' → 300000.0, '1s' → 1000.0"""
```

变换：`x → log10(max(x_ms, 1))`

适用：autovacuum_naptime, bgwriter_delay, checkpoint_timeout, lock_timeout, deadlock_timeout, idle_in_transaction_session_timeout

### 5.3 数值型 Knob → 按范围决定

| 条件 | 变换 | 示例 |
|------|------|------|
| max/min > 100x 且不含 0 | `log10(x)` | commit_delay, max_connections |
| max/min > 100x 且含 0 | `log1p(x)` | effective_io_concurrency, bgwriter_lru_maxpages |
| max/min ≤ 100x | 原值不变 | random_page_cost, cpu_*_cost, checkpoint_completion_target |

### 5.4 类别型 → 编码

| 类型 | Knob | 编码 |
|------|------|------|
| 二值 | autovacuum, track_activities, track_counts, wal_compression | `on→1, off→0` |
| 多值枚举 | synchronous_commit (on/off/local/remote_write) | LabelEncoder 或 One-hot |
| 负载类型 | workload | **One-hot**：4 列 |
| 磁盘类型 | hw_disk_type | `SSD→1, HDD→0` |

### 5.5 目标变量

```python
y = np.log1p(tps)
# 成功: log1p(11677) ≈ 9.37
# 失败: log1p(0) = 0
# 还原: tps_pred = np.expm1(model.predict(X))
```

## 六、特征归一化

### LightGBM（首选模型）

- **不需要**归一化。树模型对特征尺度不敏感。

### MLP（备选模型）

- 需要 StandardScaler 或 MinMaxScaler
- 在 log 变换之后再归一化

## 八、预处理 Pipeline 设计

```python
class KnobPreprocessor:
    """
    完整预处理 pipeline：
    raw CSV → 特征筛选 → 单位解析 → 缺失填充 → 变换 → 编码 → X, y
    """

    def __init__(self, knob_space_path: str):
        self.knob_space = load_knob_space(knob_space_path)
        self.label_encoders = {}
        self.scaler = None  # MLP 时使用

    def fit_transform(self, csv_path: str):
        df = pd.read_csv(csv_path)

        # 1. 删除无用列
        df = self._drop_columns(df)

        # 2. 解析单位
        df = self._parse_units(df)

        # 3. 填充缺失值
        df = self._fill_missing(df)

        # 4. 数值变换 (log2/log10/log1p)
        df = self._transform(df)

        # 5. 编码 (bool/enum/one-hot)
        df = self._encode(df)

        # 6. 分离 X, y
        y = np.log1p(df["tps"].astype(float))
        X = df.drop(columns=["tps", "latency_avg", "latency_p95", "status", "error"])

        return X, y, df["status"]
```

| 指标 | 目标 | 说明 |

```bash
    --data cost_model/data/raw/dataset.csv \
    --knob-space configs/knob_space.yaml \
    --output cost_model/checkpoints/v1/

python3 -m cost_model.train \
    --data cost_model/data/raw/dataset.csv \
    --knob-space configs/knob_space.yaml \
    --output cost_model/checkpoints/v1/ \
    --n-estimators 500 \
    --max-depth 8 \
```
