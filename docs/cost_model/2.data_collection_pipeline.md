# Cost Model 数据采集流程设计

## 概述

为 Cost Model 训练采集 (knob 配置, 环境特征, 性能指标) 数据集。整体流程参考 OtterTune (SIGMOD 2017)、CDBTune (SIGMOD 2019)、DBTune (VLDB 2022) 的标准做法。

## 采集流程

```
┌─────────────────────────────────────────────────┐
│                  循环 N 轮                       │
│                                                 │
│  1. Knob 生成器 → 随机生成一组 knob 配置          │
│          ↓                                      │
│  2. PG 配置器 → 写入 postgresql.conf + 重启       │
│          ↓                                      │
│  3. 快照采集 (before) → metrics_before           │
│          ↓                                      │
│  4. 负载运行器 → 跑 benchmark，记录 tps/latency   │
│          ↓                                      │
│  5. 快照采集 (after) → metrics_after             │
│          ↓                                      │
│  6. 计算 diff + 合并特征 → 追加一行到 CSV         │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 各模块设计

### 模块一：Knob 生成器

**职责**：在 knob 搜索空间内生成一组合法配置。

**采样策略**：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| 随机采样 | 各 knob 在合法范围内均匀随机 | 初期快速覆盖 |
| Latin Hypercube Sampling (LHS) | 保证各维度均匀覆盖 | 样本量有限时更高效 |
| 默认配置 | PG 默认值 | 作为 baseline |

**Knob 搜索空间定义**（YAML 配置）：

```yaml
knobs:
  shared_buffers:
    type: memory          # 内存类参数
    min: 32MB
    max: 8GB
    default: 128MB

  work_mem:
    type: memory
    min: 1MB
    max: 1GB
    default: 4MB

  effective_cache_size:
    type: memory
    min: 128MB
    max: 32GB
    default: 4GB

  random_page_cost:
    type: float
    min: 1.0
    max: 10.0
    default: 4.0

  max_parallel_workers_per_gather:
    type: integer
    min: 0
    max: 8
    default: 2

  checkpoint_completion_target:
    type: float
    min: 0.1
    max: 0.9
    default: 0.5

  # ... 其他 knob
```

### 模块二：PG 配置器

**职责**：将生成的 knob 配置应用到 PostgreSQL，并重启使之生效。

**流程**：

```
1. 备份当前 postgresql.conf
2. 修改目标 knob 值（ALTER SYSTEM SET 或直接改文件）
3. 重启 PG（pg_ctl restart）
4. 等待 PG 就绪（pg_isready 轮询）
5. 验证配置是否生效（SHOW 检查）
```

**注意事项**：
- 区分 dynamic knob（`ALTER SYSTEM SET` 后 `pg_ctl reload` 即可）和 static knob（必须 restart）
- 设置超时：如果 PG 启动失败（比如 shared_buffers 设太大），需要回滚到默认配置
- 记录每次配置变更日志

### 模块三：负载运行器

**职责**：运行 benchmark，获取性能指标。

**支持的 Benchmark**：

| Benchmark | 命令 | 输出解析 |
|-----------|------|---------|
| pgbench (OLTP) | `pgbench -c 8 -j 4 -T 60 mydb` | 解析 `tps = xxx` |
| sysbench (OLTP) | `sysbench oltp_read_write --time=60 run` | 解析 transactions/queries per sec |
| TPC-H (OLAP) | 22 条 SQL 逐条执行 | 记录每条查询耗时 |

**默认使用 pgbench**（PG 自带，零安装）：

```
初始化: pgbench -i -s 10 mydb          # scale factor 10
运行:   pgbench -c 8 -j 4 -T 60 mydb   # 8 连接，4 线程，60 秒
```

**输出解析目标**：

```
tps           # 每秒事务数
latency_avg   # 平均延迟 (ms)
latency_p95   # P95 延迟（需 --log 参数后自行计算）
```

### 模块四：快照采集器

**职责**：采集某一时刻的 DB 状态。**已实现**（`collector.py` 中的各 Collector）。

采集内容：
- 硬件特征（一次即可，不随 knob 变化）
- 软件特征（一次即可）
- Knob 配置（每轮变化）
- DB 运行时指标（每轮 before/after 各采一次）

### 模块五：Pipeline 编排

**职责**：串联上述模块，自动化循环采集。

**伪代码**：

```python
def run_pipeline(num_rounds, pg_config, benchmark_config):
    collector = DataCollector(conn)
    knob_gen = KnobGenerator("configs/knob_space.yaml")
    pg_ctl = PGConfigurator(pg_config)
    benchmark = BenchmarkRunner(benchmark_config)

    # 硬件/软件特征只采一次
    hw_features = collector.hardware_collector.collect()
    sw_features = collector.software_collector.collect(conn)

    for i in range(num_rounds):
        # 1. 生成 knob
        knobs = knob_gen.sample()

        # 2. 应用配置 + 重启
        pg_ctl.apply(knobs)

        # 3. 采集 before
        metrics_before = collector.metrics_collector.collect(conn)

        # 4. 跑负载
        perf = benchmark.run()  # → {tps, latency_avg, ...}

        # 5. 采集 after
        metrics_after = collector.metrics_collector.collect(conn)

        # 6. 组装一行 + 写入 CSV
        row = {
            **hw_features,
            **sw_features,
            **knobs,
            **diff(metrics_before, metrics_after),
            **perf,  # label
        }
        append_to_csv(row)
```

## CSV 数据格式

每一行是一个训练样本：

```
| hw_cpu_count | hw_total_memory_gb | sw_pg_version | knob_shared_buffers | knob_work_mem | metric_..._diff | tps    | latency_avg |
|-------------|-------------------|---------------|--------------------|--------------|-----------------| -------|-------------|
| 8           | 32.0              | 16.2          | 256MB              | 8MB          | 12345           | 1500.3 | 4.2         |
| 8           | 32.0              | 16.2          | 1GB                | 16MB         | 23456           | 2100.7 | 3.1         |
```

**前缀约定**：
- `hw_` — 硬件特征
- `sw_` — 软件特征
- `knob_` — Knob 配置（输入变量）
- `metric_` — DB 运行时指标差值
- 无前缀 — 性能指标（label）：`tps`、`latency_avg`、`latency_p95`

## 预计采集时间

| 参数 | 值 |
|------|---|
| 每轮 benchmark 时长 | 60s |
| PG 重启时间 | ~5s |
| 每轮总计 | ~70s |
| 采集 100 轮 | ~2 小时 |
| 采集 500 轮 | ~10 小时 |
| 采集 1000 轮 | ~20 小时 |
