# 配置文件设计

## 概述

`config.yaml` 是项目的统一配置文件，训练和推理共用。按功能模块分为多个配置段，各工具从各自的段中读取配置。

## 配置结构

```yaml
# ============================================================
#  db-opt-r1 配置文件
# ============================================================

# ==================== 通用配置 ====================
general:
  project_name: db-opt-r1
  seed: 42
  log_level: INFO                   # DEBUG / INFO / WARNING / ERROR
  log_dir: ./logs
  log_format: "%(asctime)s [%(levelname)s] %(name)s: %(message)s"

# ==================== 工具配置 ====================
tools:
  # --- 数据库连接（所有工具共用） ---
  database:
    host: 127.0.0.1
    port: 5432
    user: postgres
    password: ""
    database: postgres
    data_dir: /var/lib/postgresql/16/main    # pg_ctl restart 用
    log_path: /var/log/postgresql/postgresql-16-main.log   # get_recent_logs 用
  # --- get_recent_logs ---
  recent_logs:
    default_lines: 50               # 默认返回行数
    default_level: WARNING           # 默认最低日志级别

  # --- get_db_metrics ---
  db_metrics:
    views:                           # 要采集的 pg_stat 视图
      - pg_stat_bgwriter
      - pg_stat_archiver
      - pg_stat_database
      - pg_stat_database_conflicts
      - pg_stat_user_tables
      - pg_statio_user_tables
      - pg_stat_user_indexes
      - pg_statio_user_indexes

  # --- set_knob ---
  set_knob:
    knob_space: ./configs/knob_space.yaml   # Knob 搜索空间定义
    safety_check: true               # 是否校验 knob 值在合法范围内
    backup_before_change: true       # 修改前备份当前配置

  # --- restart_pg / reload_pg ---
  pg_control:
    restart_timeout: 30              # 重启超时(秒)
    ready_check_interval: 1          # 就绪检查间隔(秒)
    use_pg_ctl: true                 # true: pg_ctl, false: systemctl

  # --- predict_performance ---
  predict_performance:
    model_path: ./outputs/cost_model/best_model.pkl   # Cost Model 路径
    feature_config: ./configs/knob_space.yaml

  # --- run_benchmark ---
  benchmark:
    tool: pgbench                    # pgbench / sysbench
    duration: 60                     # 秒
    clients: 8
    threads: 4
    scale_factor: 10                 # pgbench -i -s 10

# ==================== Cost Model 配置 ====================
cost_model:
  model_type: xgboost               # xgboost / gp / mlp
  training:
    data_dir: ./cost_model/data/raw
    output_dir: ./outputs/cost_model
    train_ratio: 0.8
    # XGBoost 超参
    xgboost:
      n_estimators: 500
      max_depth: 8
      learning_rate: 0.05
    # Gaussian Process 超参
    gp:
      kernel: rbf
      n_restarts: 5

# ==================== Agent 训练配置 ====================
training:
  mode: grpo                         # grpo / ppo / sft
  model:
    name: Qwen/Qwen2.5-Coder-7B
    max_length: 4096
  grpo:
    num_episodes: 1000
    batch_size: 8
    group_size: 4
    learning_rate: 1e-5
    kl_coef: 0.1
  reward:
    use_cost_model: true              # true: cost model 预测, false: 真实 benchmark
    baseline: default                 # 对比基线: default / current
  sft:
    data_path: ./data/sft_data.json
    epochs: 3
  checkpoint:
    save_dir: ./checkpoints
    save_interval: 100                # 每 N episode 保存

# ==================== 评估配置 ====================
evaluation:
  use_real_benchmark: true            # 评估时强制用真实 benchmark
  benchmark_rounds: 3                 # 每组配置跑几轮取平均
  baselines:                          # 对比基线
    - default                         # PG 默认配置
    - ottertune                       # OtterTune 推荐配置
  output_dir: ./outputs/eval
```

## 配置项与工具/模块的对应关系

| 配置段 | 使用方 | 训练用 | 推理用 |
|--------|--------|:------:|:------:|
| `general` | 所有模块 | ✅ | ✅ |
| `tools.database` | 所有需连接 PG 的工具 | ✅ | ✅ |
| `tools.recent_logs` | `get_recent_logs` | ✅ | ✅ |
| `tools.db_metrics` | `get_db_metrics` | ✅ | ✅ |
| `tools.set_knob` | `set_knob` | ✅ | ✅ |
| `tools.pg_control` | `restart_pg`, `reload_pg` | ✅ | ✅ |
| `tools.predict_performance` | `predict_performance` | ✅ | ❌ |
| `tools.benchmark` | `run_benchmark`, 数据采集 | ✅ | ✅ |
| `cost_model` | Cost Model 训练/推理 | ✅ | ❌ |
| `training` | GRPO/SFT 训练 | ✅ | ❌ |
| `evaluation` | 评估脚本 | ❌ | ✅ |

## 配置加载方式

```python
import yaml

class Config:
    def __init__(self, config_path: str):
        with open(config_path) as f:
            self._config = yaml.safe_load(f)

    def get(self, key: str, default=None):
        """支持 dot notation: config.get("database.host")"""
        keys = key.split(".")
        val = self._config
        for k in keys:
            if isinstance(val, dict):
                val = val.get(k)
            else:
                return default
        return val if val is not None else default

    @property
    def tools(self): return self._config.get("tools", {})

    @property
    def database(self): return self.tools.get("database", {})

    @property
    def training(self): return self._config.get("training", {})
```
