# Backend 抽象层设计

## 背景

训练 GRPO 需要大量采样（数千个 episode），不可能每次都跑真实 benchmark（60s/轮）。需要一个抽象层让工具在**训练时用模拟环境**，**评估时用真实 PG**。

### 其他项目做法

| 项目 | 方式 | 问题 |
|------|------|------|
| CDBTune | 直接连真实 DB，在线 RL | 慢（几百轮 × 60s = 数天） |
| DBTune | 同上 | 同上 |
| GPTuner | GP surrogate 减少真实调用 | 最接近我们的思路 |
| AgentTune | LLM 减少探索轮数 | 仍需真实 DB |

**我们的核心差异**：用 Cost Model 作为环境模拟器，让 GRPO 训练成为可能。

## 架构设计

```
Tool 函数（接口不变）
      │
      ▼
   Backend（可切换）
   ┌──────────┬──────────────┐
   │          │              │
RealBackend  SimulatedBackend
（真实 PG）    （数据集 + Cost Model）
   │          │
   ├─ 采集    ├─ GRPO 训练
   └─ 评估    └─ 快速验证
```

### Backend 基类

```python
class Backend(ABC):
    # 观察类
    def get_hardware_info(self) -> dict
    def get_current_config(self, knob_names) -> dict
    def get_db_metrics(self) -> dict
    def get_workload_info(self) -> dict
    def get_recent_logs(self, lines, level) -> str

    # 行动类
    def set_knob(self, knobs) -> dict
    def restart_pg(self) -> dict
    def reload_pg(self) -> dict
    def reset_config(self) -> dict

    # 验证类
    def predict_performance(self) -> dict
    def run_benchmark(self, duration) -> dict
```

### SimulatedBackend

从数据集加载环境，用 Cost Model 预测性能。

```python
class SimulatedBackend(Backend):
    def __init__(self, dataset_path, cost_model):
        self.dataset = pd.read_csv(dataset_path)
        self.cost_model = cost_model
        self.current_env = None       # 当前环境样本
        self.current_knobs = {}       # Agent 设置的 knob
        self.baseline_tps = 0         # 默认配置的 tps

    def reset(self, sample_idx=None):
        """新 episode：随机抽一条样本作为环境"""
        row = self.dataset.iloc[sample_idx or random.randint(...)]
        self.current_env = {
            col: row[col] for col in row.index
            if col.startswith(("hw_", "sw_", "metric_"))
        }
        self.current_knobs = {
            col.replace("knob_", ""): row[col]
            for col in row.index if col.startswith("knob_")
        }
        self.baseline_tps = row["tps"]

    def get_hardware_info(self):
        # 返回 current_env 中 hw_ 开头的字段

    def get_db_metrics(self):
        # 返回 current_env 中 metric_ 开头的字段

    def set_knob(self, knobs):
        # 更新 self.current_knobs（内存操作）

    def restart_pg(self):
        return {"success": True, "duration_seconds": 0}

    def predict_performance(self):
        # 用 cost model 预测
        features = {**self.current_env, **self.current_knobs}
        return self.cost_model.predict(features)

    def run_benchmark(self, duration):
        # 训练时不可用，抛异常或返回 cost model 预测
```

用途：**GRPO 训练**。

## 工具函数改造

工具函数不再直接连 PG，而是委托给 Backend：

```python
# 改造前
def get_db_metrics() -> str:

    cursor = conn.cursor()

# 改造后
def get_db_metrics() -> str:
    return json.dumps(_backend.get_db_metrics())
```

## 训练流程

```python
# 加载数据集和 cost model
dataset = "cost_model/data/raw/dataset.csv"
cost_model = load_model("outputs/cost_model/best_model.pkl")
backend = SimulatedBackend(dataset, cost_model)

for episode in range(num_episodes):
    backend.reset()                       # 随机环境
    env = DBToolEnv(backend, max_turns=10)

    # Agent 交互
    prompt = build_prompt(env)
    for turn in range(max_turns):
        action = agent.generate(prompt)
        obs, reward, done, info = step(env, action)
        if done: break

    # GRPO 更新
    trainer.update(...)
```

## Reward 设计

```python
reward = (predicted_tps - baseline_tps) / baseline_tps
```

- `baseline_tps`：数据集中该样本的原始 tps（默认配置下的性能）
- `predicted_tps`：Agent 调整 knob 后 cost model 预测的 tps
- reward > 0 表示 Agent 找到了更好的配置
