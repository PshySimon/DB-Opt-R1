# db-opt-r1 目录架构设计

## 项目定位

基于 LLM Agent + GRPO + Cost Model 的数据库参数调优系统。

## 目录结构

```
db-opt-r1/
├── docs/                       # 项目文档
│   ├── research/               # 调研文档
│   ├── architecture/           # 架构设计文档
│   ├── cost_model/             # Cost Model 相关文档
│   ├── training/               # 训练相关文档
│   └── evaluation/             # 评估相关文档
│
├── core/                       # 通用框架层
│   ├── trainer/                # RL 训练引擎
│   ├── agent/                  # LLM Agent 多轮交互引擎
│   ├── tool/                   # 工具调用框架
│   └── data/                   # 数据集加载与处理
│
├── environment/                # 数据库调优环境
│   └── tools/                  # Agent 可调用的工具实现
│
├── cost_model/                 # Cost Model 模块
│   ├── models/                 # 模型实现（XGBoost、GP 等）
│   └── data/                   # 训练数据（采集、处理）
│
├── configs/                    # 配置文件
├── scripts/                    # 运行脚本
├── tests/                      # 测试
└── third_party/                # 第三方依赖
```

## 各目录职责

### `core/` — 通用框架层

与具体调优任务无关，可复用的基础设施。主要从 Agent-R1 的 verl 框架中提炼。

| 子目录 | 职责 |
|--------|------|
| `trainer/` | GRPO 训练循环、Ray 分布式编排、Reward 管理。从 Agent-R1 的 `agent_ray_trainer.py` 中抽取通用部分 |
| `agent/` | LLM 的多轮工具调用管理。负责生成 → 解析 tool_call → 执行工具 → 拼接结果 → 继续生成 的循环。从 Agent-R1 的 `ToolGenerationManager` 中复用 |
| `tool/` | 工具的抽象基类、工具环境（step/reset）、工具注册机制。定义 Tool 接口供 `environment/tools/` 实现 |
| `data/` | RL 训练数据集的加载和预处理逻辑 |

### `environment/` — 数据库调优环境

Agent 与数据库交互的一切逻辑。对应 RL 中的 "environment" 概念。

| 子目录 | 职责 |
|--------|------|
| 根目录 | DB 连接管理、knob 定义与约束、workload 运行、reward 计算 |
| `tools/` | Agent 可调用的具体工具实现，如：查看当前配置、应用新配置、运行 benchmark、查看运行指标等 |

### `cost_model/` — Cost Model 模块

替代真实数据库执行的代理模型，训练时提供快速 reward 反馈。

| 子目录 | 职责 |
|--------|------|
| `models/` | 各种 cost model 实现（XGBoost、Gaussian Process 等）及其训练逻辑 |
| `data/` | 数据采集脚本（参考 OtterTune 的 pg_stat 视图采集）、原始数据存储、数据清洗与特征工程 |

### `configs/` — 配置文件

训练超参、数据库连接信息、knob 搜索空间定义、workload 配置等。

### `scripts/` — 运行脚本

各阶段的入口脚本：数据采集、cost model 训练、Agent SFT、GRPO 训练、真实环境评估。

### `tests/` — 测试

各模块的单元测试和集成测试。

### `third_party/` — 第三方依赖

verl（RL 训练框架）等外部依赖。

## 数据流

```
                        ┌─────────────┐
                        │  cost_model │
                        │    /data    │ ← 真实 PG 采集
                        └──────┬──────┘
                               │ 训练
                        ┌──────▼──────┐
                        │  cost_model │
                        │   /models   │ ← XGBoost 等
                        └──────┬──────┘
                               │ 预测 reward
┌──────────┐  prompt   ┌──────▼──────┐  tool_call  ┌─────────────┐
│ core/data├──────────►│ core/agent  ├────────────►│ environment │
└──────────┘           │  (LLM)     │◄────────────┤   /tools    │
                       └──────┬──────┘  observe    └─────────────┘
                              │ loss
                       ┌──────▼──────┐
                       │core/trainer │
                       │   (GRPO)    │
                       └─────────────┘
```
